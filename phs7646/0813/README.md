그리디 : 정렬 순서와 반대로 매칭하면 손해
dp[a][b] : 많은쪽에서 a번까지 사용해서 적은쪽에서 b번까지 매칭했을 때 성격차이의 최소값
dp[a][b] = min(dp[a-1][b], dp[a-1][b-1] + cost(a<->b))  : a와 b를 매칭하는 경우와 하지 않는 경우
